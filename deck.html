Mistral Agent with Autonomous Tool Use

The speaker resolution agent runs a multi-turn loop with mistral-large-latest and tool_choice: auto. It reasons over both acoustic evidence (voiceprint similarity) and semantic cues (self-introductions, role references, attendee context).

Speaker Memory

zvec: In-Memory Vector Store for Voiceprints

Most meeting tools treat every call as a blank slate. MMAGA remembers voices. Enroll a speaker once, and the system matches them acoustically in every future upload.

Why zvec?

  • Embedded in-process — no external database to provision or manage
  • Native FP32 cosine similarity over 192-dim vectors
  • Sub-millisecond top-k lookup for voiceprint matching
  • Persistent storage with zero-config initialization
  • Hackathon-deployable: one container, no infra overhead

Novel Speaker Identification

zvec enables a pipeline that most meeting tools can't offer: extract a speaker embedding from a ~5s audio segment, query the store for cosine-similar enrolled voices, and feed ranked candidates as evidence into the Mistral agent. The agent fuses acoustic similarity with semantic cues to make the final attribution.

Enrollment Flow

Audio → GPU /embed → 192-d vector → zvec store

Identification Flow

Segment Selection Algorithm

For each speaker, the orchestrator selects an optimal ~5s non-overlapping segment. Falls back to adjacent-segment concatenation (gap ≤ 0.5s) to ensure clean embeddings without speaker interference.

Future Vision

Where MMAGA Goes Next

FUTURE 01

Voxtral for Semantic Moment Detection

Voxtral Small (24B) can answer questions about audio directly: "What was the most contested claim?" or "Did anyone express doubt about the Q3 deadline?" — without transcribing first.

Use this as a pre-processing pass to identify which segments need deep LLM analysis versus which can be summarized cheaply. Query the audio like a database, not just transcribe it.

FUTURE 02

The "Dissent Surface"

Every meeting tool surfaces consensus. Nobody surfaces dissent. Detect hedged language, contradiction, and soft disagreement ("I guess that could work") versus firm commitment ("Absolutely, I'll own that").

Render a visual Dissent Map: a heatmap of the conversation timeline where disagreement intensity is color-coded. Surface what every other tool hides.

FUTURE 03

Retroactive Clarification via Async Voice

Instead of interrupting live meetings with AI clarification, flip it: after the meeting, generate voice questions (TTS) and send them as voice messages to the relevant person via Slack or email. "Hey Marcus, it sounded like you were taking on the API migration — can you confirm and set a deadline?" The response feeds back and resolves the minutes. Less impressive as a live demo, far more production-realistic.

Future Vision

Cross-Meeting Intelligence & UX

FUTURE 04

Pre-Mortem: Meeting Debt Detection

After generating action items, query a vector-indexed store of past meetings to find semantically similar past commitments: "This task resembles 'fix the auth flow' from March 14 — that one was never resolved."

Surface this as a warning: "Similar commitment made 6 weeks ago. Status unknown." Cross-meeting memory transforms minutes from a snapshot into a living accountability system.

FUTURE 05

UX Beyond AI Defaults

Design principles for the next iteration:

  • Transcript as timeline — horizontal speaker lanes like a multi-track editor. Ambiguities pulse. Resolved items turn green.
  • Decision ledger — who proposed, who seconded, who dissented, final status. Inspired by git commit history.
  • Clarification as structured evidence — "AMBIGUITY DETECTED. Referential. Line 14:32. Subject unresolved." Two buttons: Confirm or Correct.

These features build on the foundation we've shipped: the voiceprint memory, the agentic tool loop, and the structured output schema are all extensible by design.
